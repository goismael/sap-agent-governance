# Contributing to SAP Agent Governance Patterns

Contributions are welcome. This library grows through the experience of practitioners who have built, broken, and fixed agentic AI systems on SAP.

## What Makes a Good Pattern

A pattern belongs here if it:
- Addresses a real, recurring governance challenge in SAP-integrated agent systems
- Is grounded in how SAP authorization, data, or control frameworks actually work
- Includes concrete implementation guidance — not just principles
- Addresses what auditors or compliance teams will ask

## How to Contribute

1. Fork the repository
2. Create a branch: `pattern/P00X-short-description`
3. Use the pattern template in `docs/PATTERN_TEMPLATE.md`
4. Submit a pull request with a description of the real problem your pattern solves

## What This Library Is Not

- Not a general SAP security guide
- Not a vendor product recommendation
- Not a compliance checklist (though it informs one)
- Not derived from any specific client engagement or proprietary methodology
